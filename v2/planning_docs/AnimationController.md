# AnimationController

Central API for all rider animation - procedural dynamics, polish animations, and tricks.

## Architecture

```
PlayerEntity
├── VisualRoot (Node3D)           # Rotates for lean + pitch (bike + rider together)
│   ├── CharacterSkin
│   │   ├── IKController          # FABRIK solver, holds @export refs to body markers
│   │   ├── IKAnimationPlayer     # IK marker animations (polish/tricks)
│   │   ├── AnimationPlayer       # Skeleton animations
│   │   └── IKTargets/            # Character-owned body-shape markers
│   │       ├── ChestTarget
│   │       ├── HeadTarget
│   │       ├── LeftArmMagnet / RightArmMagnet
│   │       └── LeftLegMagnet / RightLegMagnet
│   └── BikeSkin                  # Owns contact-point markers
│       ├── SeatMarker            # → IKController.butt_pos (at runtime)
│       ├── LeftHandleBarMarker   # Source for steering proxy
│       ├── SteeringHandleBarProxy # (runtime) child of RotationPoint, follows steering
│       ├── RightHandProxy        # (runtime) mirrored from steering proxy
│       ├── LeftPegMarker         # → IKController.ik_left_foot (at runtime)
│       ├── RightFootProxy        # (runtime) mirrored from left peg
│       ├── FrontWheelGroundMarker
│       ├── RearWheelGroundMarker
│       ├── FrontWheelFrontMarker # Front of front tire (stoppie pivot arc)
│       └── RearWheelBackMarker   # Back of rear tire (wheelie pivot arc)
│
└── _Controllers
    ├── AnimationController
    ├── MovementController
    ├── InputController
    └── CameraController
```

### Marker Ownership

**Bike owns** contact points (in BikeSkin scene, saved to BikeSkinDefinition):
- `SeatMarker` — rider's butt position
- `LeftHandleBarMarker` — hand grip position (a runtime proxy under RotationPoint follows steering)
- `LeftPegMarker` — foot peg position
- Right side generated by mirroring X at runtime

**Character owns** body-shape markers (in CharacterSkin/IKTargets, set via @export):
- `ChestTarget`, `HeadTarget`
- `LeftArmMagnet`, `RightArmMagnet`
- `LeftLegMagnet`, `RightLegMagnet`

**IKController wiring:** `PlayerEntity._init_ik()` passes bike marker node references to IKController at runtime via `set_bike_markers()`. Contact-point markers (butt, hands, feet) are runtime vars, not @exports.

### Per-Bike Rider Pose

Body-shape marker positions are stored on `BikeSkinDefinition` under `@export_group("Rider Pose")`:
```
chest_position, chest_rotation
head_position, head_rotation
left_arm_magnet_position, right_arm_magnet_position
left_leg_magnet_position, right_leg_magnet_position
```

These are applied at init by `PlayerEntity._apply_rider_pose_from_definition()` and by the editor "Init IK from Bike" / "Reset to Default Pose" buttons.

### Steering — Hands Follow Automatically

`BikeSkin._create_steering_handlebar_proxy()` creates a `SteeringHandleBarProxy` Marker3D as a child of the mesh's `RotationPoint`. When `rotate_steering()` rotates RotationPoint, the proxy (and thus the hand IK target) moves with it. The right-side proxy mirrors from the left proxy's transform. No manual hand-steering math needed.

## State Machine

```gdscript
enum RiderState {
    RIDING,   # Procedural active, IK enabled
    IDLE,     # Procedural paused, IK animations playing
    TRICK,    # IK disabled, skeleton animation playing
    RAGDOLL,  # Everything disabled
}
```

## Public API

```gdscript
# Procedural control (automatic in RIDING state)
func set_procedural_enabled(enabled: bool)

# Polish animations (IK marker anims)
func play_idle_animation(anim_name: String)
func play_land_settle()

# Trick animations (skeleton anims)
func play_trick(trick_name: String)

# Ragdoll
func start_ragdoll()
func stop_ragdoll()

# Setup (called by PlayerEntity after IK init)
func initialize()
```

## Rotation Layers

Two independent rotation systems on `visual_root`:

| Layer | Axis | Controls | Example |
|-------|------|----------|---------|
| Lean | `visual_root.rotation.z` | Bike + rider tilt into turns | Cornering lean |
| Pitch | `visual_root.rotation.x` | Wheelies, stoppies | Wheelie/stoppie visual |

Both are driven procedurally from `movement_controller` state.

---

## Wheelie/Stoppie Pivot

The pivot point lerps along the tire's contact arc based on pitch angle:

```gdscript
var pitch_ratio = clamp(abs(visual_root.rotation.x) / (PI / 2.0), 0.0, 1.0)
# Wheelie: rear_wheel_ground → rear_wheel_back
# Stoppie: front_wheel_ground → front_wheel_front
pivot = ground_position.lerp(edge_position, pitch_ratio)
```

At 0° the pivot is the bottom of the wheel. At 90° it's the back (or front) of the tire. This prevents the bike from clipping through the ground during deep wheelies.

## IK System

### How It Works

1. **Bike markers** set by `PlayerEntity._init_ik()` — passes bike contact points to IKController via `set_bike_markers()`
2. **Rider pose** applied from `BikeSkinDefinition` fields (chest, head, magnets)
3. **FABRIK3D** solves bone positions to reach marker targets
4. **IKController** applies marker rotations to end bones (FABRIK only handles position)
5. **AnimationController** applies procedural offsets on top of base positions

### IK Markers

**Character-owned** (in `CharacterSkin/IKTargets`, wired via @export):

| Node name | GDScript ref | Controls |
|-----------|-------------|----------|
| `ChestTarget` | `ik_chest` | Spine bone rotation |
| `HeadTarget` | `ik_head` | Head bone rotation |
| `LeftArmMagnet` / `RightArmMagnet` | `ik_left_arm_magnet` / `ik_right_arm_magnet` | Elbow position |
| `LeftLegMagnet` / `RightLegMagnet` | `ik_left_leg_magnet` / `ik_right_leg_magnet` | Knee position |

**Bike-owned** (set at runtime via `set_bike_markers()`):

| Source | GDScript ref | Controls |
|--------|-------------|----------|
| `BikeSkin.seat_marker` | `butt_pos` | Hips bone position |
| `BikeSkin.steering_handlebar_marker` | `ik_left_hand` | Left hand position + rotation |
| Runtime mirror proxy | `ik_right_hand` | Right hand position + rotation |
| `BikeSkin.left_peg_marker` | `ik_left_foot` | Left foot position + rotation |
| Runtime mirror proxy | `ik_right_foot` | Right foot position + rotation |

### Bone-to-Marker Mapping

```
Hips         <- butt_pos (position only)
Spine        <- ik_chest (rotation)
Head         <- ik_head (rotation)
LeftHand     <- ik_left_hand (position + rotation)
RightHand    <- ik_right_hand (position + rotation)
LeftFoot     <- ik_left_foot (position + rotation)
RightFoot    <- ik_right_foot (position + rotation)
```

## Creating IK Animations

IK animations animate the **character-owned markers** (chest, head, magnets), not the skeleton directly. Contact-point markers (hands, feet, butt) are owned by the bike and should not be keyframed in IK animations.

### Step 1: Open the Scene

Open `player_entity.tscn` to see the character on the bike. The IK markers are under `CharacterSkin/IKTargets`.

### Step 2: Create Animation in IKAnimationPlayer

1. Select `IKAnimationPlayer` node
2. In the Animation panel: **Animation > New**, name it, save it **under `IK_anim_lib`**
3. Add tracks for character-owned marker transforms:
   - `IKTargets/ChestTarget:position` / `IKTargets/ChestTarget:rotation`
   - `IKTargets/HeadTarget:position` / `IKTargets/HeadTarget:rotation`
   - `IKTargets/LeftArmMagnet:position`, `IKTargets/RightArmMagnet:position`
   - `IKTargets/LeftLegMagnet:position`, `IKTargets/RightLegMagnet:position`

### Step 3: Keyframe Marker Positions/Rotations

Animate markers by moving them in the viewport or setting values in the inspector.

**Tips:**
- Magnets control bend direction (elbow/knee)
- `ChestTarget` controls torso lean/twist
- Keep movements subtle for polish anims

### Step 4: Play via AnimationController

```gdscript
animation_controller.play_idle_animation("idle_fidget")
```

## Creating Trick Animations (Skeleton)

For complex tricks that can't be achieved with IK, use skeleton animations.

### Step 1: Create Animation in AnimationPlayer

1. Select `AnimationPlayer` node in `CharacterSkin`
2. Create animation (e.g., "trick_superman")
3. Add bone transform tracks directly

### Step 2: Play via AnimationController

```gdscript
animation_controller.play_trick("trick_superman")
```

This **disables IK** and plays the skeleton animation directly. When done:
```gdscript
animation_controller.cancel_trick()  # Returns to RIDING state, re-enables IK
```

## Procedural Animation Details

Runs automatically in `RIDING` state. Driven by input/movement:

| Effect | Input Source | Target |
|--------|--------------|--------|
| Lean (rider) | `movement_controller.roll_angle` | `butt_pos.x`, `ik_chest.rotation.y` |
| Lean (visual) | `movement_controller.roll_angle` | `visual_root.rotation.z` |
| Weight shift | `input_controller.nfx_lean` | `butt_pos.z`, `ik_chest.position.z` |
| Pitch | `movement_controller.pitch_angle` | `visual_root.rotation.x` (with pivot offset) |
| Steering | `movement_controller.roll_angle` | `bike_skin.rotate_steering()` → hands follow automatically |

### Tuning

In `BikeSkinDefinition`:
```gdscript
@export var lean_multiplier: float = 1.0
@export var weight_shift_multiplier: float = 1.0
```

In `AnimationController` inspector:
- `max_butt_offset` — Max lateral butt shift when leaning
- `max_chest_lean_pitch_deg` — Max chest pitch when leaning fwd/back
- `max_chest_z_offset` — Max chest Z shift when leaning fwd/back
- `max_butt_z_offset` — Max butt Z shift when leaning fwd/back

## Editor Workflow (Authoring from player_entity.tscn)

Open `player_entity.tscn` to author animations with the character on the bike.

### Setup (once per bike)
1. Select `AnimationController` in the scene tree
2. Click **"Init IK from Bike"** — positions IK targets from bike markers, loads rider pose from BikeSkinDefinition
3. Manually adjust marker positions/rotations in the viewport (chest angle, magnet positions, etc.)
4. Click **"Save Default Pose"** — saves chest/head/magnet transforms to the `BikeSkinDefinition` resource

### Restoring
1. Click **"Play Default Pose"** — reads rider pose fields from BikeSkinDefinition and applies them to character markers

### Authoring each animation (e.g. a trick)
1. Click **"Play Default Pose"** — restores markers to the saved base
2. Select `IKAnimationPlayer` in the scene tree
3. In the Animation panel: **Animation > New** — name it and save it **under `IK_anim_lib`**
4. At **time=0**, keyframe `position` and `rotation` for all markers you intend to animate
5. Scrub to **time=1**, move/rotate the desired markers to the trick pose, then update those keyframes

> **Why keyframe at t=0?** The `IKAnimationPlayer` is seeked by ratio (e.g. `wheelie_arm_drag` is seeked 0→1 based on pitch). Without t=0 keyframes the animation has no defined start, causing snapping when it first plays. Always bookend both ends.

## Netfox Considerations

- Procedural animation is **cosmetic-only** (doesn't affect physics)
- Runs locally on each client, no sync needed
- Input/movement state already synced, procedural derives from that
- Tricks affecting physics would need the `rb_*` pattern from CLAUDE.md
